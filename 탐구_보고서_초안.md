# 이음(Ieum) 탐구 보고서 초안

## 1. 탐구 주제

한국어 키워드 기반 프로그래밍 언어 이음(Ieum) 구현

## 2. 탐구 동기

프로그래밍 언어는 대부분 영어 키워드를 사용한다. 예를 들어 `if`, `for`, `return` 같은 단어는 많은 언어에서 공통적으로 사용된다. 이 탐구에서는 같은 개념을 한국어 키워드로 표현할 수 있는지 확인하고, 프로그래밍 언어가 실행되는 내부 과정을 직접 구현해보고자 했다.

## 3. 탐구 목표

- 한국어 키워드를 사용하는 간단한 언어를 설계한다.
- 소스 코드를 토큰으로 나누는 렉서를 구현한다.
- 토큰 목록을 AST로 바꾸는 파서를 구현한다.
- AST를 실행하는 트리워킹 인터프리터를 구현한다.
- 예제와 테스트를 통해 구현 결과를 검증한다.

## 4. 구현 환경

- 언어: Python 3.10 이상
- 방식: 트리워킹 인터프리터
- 외부 라이브러리: 사용하지 않음
- 실행 파일 확장자: `.ieum`

## 5. 언어 문법 설계

| 개념 | 이음 키워드 또는 문법 |
| --- | --- |
| 변수 선언 | `변수` |
| 함수 선언 | `함수` |
| 반환 | `반환` |
| 조건문 | `만약`, `아니면` |
| 반복문 | `반복` |
| 참/거짓 | `참`, `거짓` |
| 값 없음 | `없음` |
| 논리 연산 | `그리고`, `또는`, `아니다` |

예시:

```ieum
함수 더하기(a, b) {
    반환 a + b
}

출력(더하기(2, 3))
```

## 6. 구현 과정

### 6.1 토큰 정의

`tokens.py`에서 키워드, 리터럴, 연산자, 구분자 토큰을 정의했다. 렉서는 이 토큰 정의를 기준으로 소스 코드를 분석한다.

### 6.2 렉서

`lexer.py`는 소스 코드를 왼쪽에서 오른쪽으로 읽으며 토큰 목록을 만든다. 숫자, 문자열, 한글 식별자, 주석, 연산자를 처리한다.

### 6.3 AST 노드

`ast_nodes.py`는 파서와 실행기 사이에서 사용하는 중간 구조를 정의한다. 예를 들어 `Binary`, `Literal`, `Variable`, `IfStmt`, `FunctionStmt` 같은 노드가 있다.

### 6.4 파서

`parser.py`는 토큰 목록을 AST로 변환한다. 연산자 우선순위를 반영하기 위해 대입, 논리 연산, 비교, 덧셈/뺄셈, 곱셈/나눗셈, 단항 연산 순서로 파싱한다.

### 6.5 실행기

`evaluator.py`는 AST를 직접 순회하며 실행한다. 변수는 `environment.py`를 통해 관리하며, 함수 호출과 반환도 처리한다.

## 7. 실험 및 검증

예제 파일을 통해 다음 기능을 확인했다.

- `hello.ieum`: 문자열 출력
- `arithmetic.ieum`: 산술 연산
- `conditional.ieum`: 조건문
- `loop.ieum`: 반복문
- `function.ieum`: 함수 선언과 호출

테스트 파일을 통해 다음을 검증했다.

- 렉서가 올바른 토큰을 생성하는지
- 파서가 올바른 AST 노드를 생성하는지
- 실행기가 올바른 출력 결과를 만드는지
- 잘못된 코드에서 오류가 발생하는지

## 8. 탐구 결과

한국어 키워드만으로도 변수, 조건문, 반복문, 함수 같은 기본 프로그래밍 구조를 표현할 수 있었다. 또한 렉서, 파서, AST, 실행기가 서로 연결되면서 하나의 언어가 동작하는 과정을 확인할 수 있었다.

## 9. 한계

- 문장 구분 방식이 아직 더 명확해질 필요가 있다.
- 오류 메시지가 초보자에게 충분히 친절한지 추가 검토가 필요하다.
- 배열, 딕셔너리, 입력 함수 같은 기능은 아직 없다.
- 표준 라이브러리가 없어 복잡한 프로그램 작성에는 한계가 있다.

## 10. 개선 방향

- 줄바꿈 또는 세미콜론 기반 문장 구분 규칙 확정
- 사용자 입력 함수 추가
- 리스트 자료형 추가
- AST 출력 도구 추가
- 더 많은 예제와 통합 테스트 작성

## 11. 느낀 점

프로그래밍 언어는 단순히 문법만 있는 것이 아니라, 소스 코드를 단계적으로 해석하고 실행하는 여러 구성 요소가 연결되어 동작한다는 점을 알 수 있었다. 특히 한국어 키워드를 사용하면서도 기존 프로그래밍 언어의 구조를 유지할 수 있다는 점이 흥미로웠다.
