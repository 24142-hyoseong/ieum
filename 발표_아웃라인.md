# 이음(Ieum) 발표 아웃라인

## 1. 발표 제목

한국어 키워드 기반 프로그래밍 언어 이음(Ieum) 만들기

## 2. 발표 목적

- 프로그래밍 언어가 실행되는 기본 과정을 이해한다.
- 한국어 키워드를 사용하는 간단한 언어를 직접 설계한다.
- 렉서, 파서, AST, 인터프리터의 역할을 설명한다.

## 3. 문제의식

대부분의 프로그래밍 언어는 영어 키워드를 중심으로 설계되어 있다. 이 프로젝트는 한국어 키워드로도 프로그래밍 언어의 핵심 구조를 표현할 수 있는지 탐구한다.

## 4. 구현 방식

- Python 3.10 이상 사용
- 외부 라이브러리 없이 구현
- 트리워킹 인터프리터 방식
- 파일 확장자 `.ieum`

## 5. 언어 처리 흐름

```text
소스 코드(.ieum)
  -> Lexer
  -> Token 목록
  -> Parser
  -> AST
  -> Evaluator
  -> 실행 결과
```

## 6. 주요 파일 설명

| 파일 | 역할 |
| --- | --- |
| `tokens.py` | 토큰 종류와 키워드 정의 |
| `lexer.py` | 소스 코드를 토큰으로 변환 |
| `ast_nodes.py` | AST 노드 구조 정의 |
| `parser.py` | 토큰 목록을 AST로 변환 |
| `environment.py` | 변수 스코프 관리 |
| `evaluator.py` | AST 실행 |
| `main.py` | `.ieum` 파일 실행 |

## 7. 지원 문법 시연

1. 문자열 출력
2. 산술 연산
3. 변수 선언과 대입
4. 조건문
5. 반복문
6. 함수 선언과 호출

## 8. 시연 순서

```bash
python main.py examples/hello.ieum
python main.py examples/arithmetic.ieum
python main.py examples/conditional.ieum
python main.py examples/loop.ieum
python main.py examples/function.ieum
```

## 9. 테스트 설명

- 렉서 테스트: 키워드, 식별자, 숫자, 문자열, 연산자, 주석 처리 확인
- 파서 테스트: AST 노드 구조 확인
- 실행기 테스트: 출력 결과와 런타임 오류 확인

## 10. 어려웠던 점

- 한국어 키워드와 일반 식별자를 구분하는 과정
- 연산자 우선순위를 파서에 반영하는 과정
- 함수의 `반환`을 실행 흐름으로 처리하는 과정
- 변수 스코프를 블록과 함수에서 올바르게 관리하는 과정

## 11. 한계와 개선 방향

- 문장 구분 규칙을 더 명확히 해야 한다.
- 오류 메시지를 더 친절하게 만들 수 있다.
- 리스트, 딕셔너리 같은 자료구조를 추가할 수 있다.
- 디버깅용 AST 출력 기능을 만들 수 있다.

## 12. 마무리

이음은 작은 언어이지만, 프로그래밍 언어가 코드를 읽고 실행하는 전체 흐름을 직접 구현해볼 수 있는 탐구 결과물이다.
